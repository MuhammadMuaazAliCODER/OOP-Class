Basics of Class and Object

A class is a blueprint/template. An object is an instance of a class.

Key Terms

  • Class: User-defined data type
  • Object: Instance of a class
  • Data members: Variables inside class
  • Member functions: Functions inside class
  • Access specifiers: public, private, protected

Access Specifiers

SpecifierAccessible From
publicAnywhere
privateSame class only
protectedClass + subclasses

Basic Class Syntax

class Student {
private:
    string name;    // data member
    int age;

public:
    // Member function
    void setName(string n) { name = n; }
    string getName() { return name; }
    void display() {
        cout << "Name: " << name << ", Age: " << age << endl;
    }
};

int main() {
    Student s1;           // object creation
    s1.setName("Ali");
    s1.display();
    
    Student* ptr = new Student(); // dynamic object
    ptr->setName("Sara");
    delete ptr;
}

this Pointer

The this pointer points to the current object. Used to disambiguate member variables from parameters.

class Box {
    int width;
public:
    void setWidth(int width) {
        this->width = width; // this->width = member, width = parameter
    }
};
⚠️ Default access in class is private. In struct, it's public.

Constructors, Copy & Initializer Lists

A constructor is a special function called automatically when an object is created. Same name as class, no return type.

Types of Constructors

TypeParametersWhen Called
DefaultNoneStudent s;
ParameterizedOne or moreStudent s("Ali", 20);
Copyconst ClassName&Student s2 = s1;
class Student {
private:
    string name;
    int age;
public:
    // Default Constructor
    Student() {
        name = "Unknown";
        age = 0;
        cout << "Default constructor called\n";
    }
    
    // Parameterized Constructor
    Student(string n, int a) {
        name = n;
        age = a;
        cout << "Parameterized constructor called\n";
    }
    
    // Copy Constructor
    Student(const Student& other) {
        name = other.name;
        age = other.age;
        cout << "Copy constructor called\n";
    }
    
    void display() { cout << name << " " << age << endl; }
};

int main() {
    Student s1;                 // Default
    Student s2("Ali", 20);      // Parameterized
    Student s3(s2);             // Copy
    Student s4 = s2;            // Also copy constructor!
}

Initializer Lists

✅ Preferred way to initialize members — more efficient, required for const and reference members.
class Point {
private:
    int x, y;
    const int ID; // must use initializer list!
public:
    // Initializer list syntax: constructor() : member(value) {}
    Point(int x, int y, int id) : x(x), y(y), ID(id) {
        // body can be empty or do additional work
    }
    
    void display() { cout << "(" << x << "," << y << ")" << endl; }
};

Shallow vs Deep Copy

⚠️ Shallow copy copies pointer addresses — both objects share the same memory → CRASH on double delete!
// SHALLOW COPY (problematic with dynamic memory)
class ShallowDemo {
public:
    int* data;
    ShallowDemo(int val) { data = new int(val); }
    // Default copy constructor does shallow copy
    // Both objects point to SAME memory — DANGEROUS
};

// DEEP COPY (correct solution)
class DeepDemo {
public:
    int* data;
    DeepDemo(int val) { data = new int(val); }
    
    // Custom copy constructor — allocates NEW memory
    DeepDemo(const DeepDemo& other) {
        data = new int(*other.data); // new memory, same value
    }
    
    ~DeepDemo() { delete data; } // safe: each deletes its own
};

Pointer copiedAddress onlyNew memory allocated
Independence❌ Shared memory✅ Independent
RiskDouble delete crashSafe
When neededNo dynamic memoryHas dynamic memory

Destructor & Dynamic Memory

Destructor

Called automatically when an object goes out of scope or is deleted. Used to free dynamic memory.

Destructor name = ~ClassName() | No return type | No parameters | One per class
class DynArray {
private:
    int* arr;
    int size;
public:
    DynArray(int n) {
        size = n;
        arr = new int[n];
        cout << "Constructor: memory allocated\n";
    }
    
    ~DynArray() {
        delete[] arr;  // Free array memory
        cout << "Destructor: memory freed\n";
    }
    
    void set(int i, int val) { arr[i] = val; }
    int get(int i) { return arr[i]; }
};

new and delete

OperationSyntaxNotes
Allocate singlenew Type(val)Returns pointer
Allocate arraynew Type[n]Array on heap
Free singledelete ptrFrees 1 object
Free arraydelete[] ptrMust use [] for arrays!
int* p = new int(42);  // single int on heap
delete p;
p = nullptr;           // good practice!

int* arr = new int[5]; // array on heap
arr[0] = 10;
delete[] arr;          // MUST use delete[]
⚠️ Common mistakes: Using delete instead of delete[] for arrays | Not setting pointer to nullptr after delete | Forgetting to delete (memory leak)

Full Example — Rule of Three

If you define any of: destructor, copy constructor, copy assignment — define all three.

class MyString {
private:
    char* str;
public:
    MyString(const char* s) {   // Constructor
        str = new char[strlen(s)+1];
        strcpy(str, s);
    }
    MyString(const MyString& o) { // Copy Constructor
        str = new char[strlen(o.str)+1];
        strcpy(str, o.str);
    }
    MyString& operator=(const MyString& o) { // Copy Assignment
        if (this != &o) {
            delete[] str;
            str = new char[strlen(o.str)+1];
            strcpy(str, o.str);
        }
        return *this;
    }
    ~MyString() { delete[] str; }  // Destructor
};

Encapsulation

Encapsulation = bundling data + functions together, restricting direct access via access specifiers. Data is hidden, accessed via getters/setters.

✅ Benefits: Data protection | Controlled access | Easier maintenance | Data validation
class BankAccount {
private:
    double balance;   // hidden from outside world
    string owner;
    
    bool isValidAmount(double amt) {  // private helper
        return amt > 0;
    }
    
public:
    BankAccount(string name, double initial)
        : owner(name), balance(initial) {}
    
    // Getter — read only access
    double getBalance() const { return balance; }
    
    // Controlled setter with validation
    bool deposit(double amount) {
        if (isValidAmount(amount)) {
            balance += amount;
            return true;
        }
        cout << "Invalid deposit amount!\n";
        return false;
    }
    
    bool withdraw(double amount) {
        if (isValidAmount(amount) && amount <= balance) {
            balance -= amount;
            return true;
        }
        cout << "Insufficient funds or invalid amount!\n";
        return false;
    }
};

int main() {
    BankAccount acc("Ali", 1000);
    // acc.balance = 5000; // ERROR! private
    acc.deposit(500);
    cout << acc.getBalance(); // 1500
}

Member Functions & Friend Functions

Member Functions

Functions defined inside or associated with a class. Can be defined inside class (inline) or outside using ::.

class Circle {
private:
    double radius;
public:
    Circle(double r) : radius(r) {}
    
    double area();        // declaration inside
    double perimeter();  // declaration inside
};

// Definition outside using scope resolution ::
double Circle::area() {
    return 3.14159 * radius * radius;
}
double Circle::perimeter() {
    return 2 * 3.14159 * radius;
}

Friend Function

A friend function is NOT a member of the class, but has access to ALL private/protected members. Declared inside class with friend keyword.
class Box {
private:
    double volume;
public:
    Box(double v) : volume(v) {}
    
    // Friend function declaration
    friend double getVolume(Box& b);
    friend void compareBoxes(Box& b1, Box& b2);
};

// Friend function definition (NOT part of Box class)
double getVolume(Box& b) {
    return b.volume; // CAN access private member
}

void compareBoxes(Box& b1, Box& b2) {
    if (b1.volume > b2.volume)
        cout << "Box 1 is bigger";
    else
        cout << "Box 2 is bigger";
}

Pass by Value vs Pass by Reference

// PASS BY VALUE — copy is made
void doubleVal(int x) {
    x *= 2;  // original unchanged
}

// PASS BY REFERENCE — original is modified
void doubleRef(int& x) {
    x *= 2;  // original IS changed
}

// PASS BY CONST REF — efficient, no copy, no modification
void printStudent(const Student& s) {
    s.display(); // read only, no copy overhead
}

int main() {
    int n = 10;
    doubleVal(n);  cout << n; // 10 (unchanged)
    doubleRef(n);  cout << n; // 20 (changed)
}
MethodSyntaxOriginal Changed?Copies Data?
By Valuevoid f(int x)NoYes
By Referencevoid f(int& x)YesNo
By Const Refvoid f(const int& x)NoNo

Operator Overloading

Giving custom meaning to operators for user-defined types. Use operator keyword.

+ and - Operators ▼
class Vector {
public:
    int x, y;
    Vector(int x=0, int y=0) : x(x), y(y) {}
    
    // + operator: returns new Vector (doesn't modify)
    Vector operator+(const Vector& other) const {
        return Vector(x + other.x, y + other.y);
    }
    
    // - operator
    Vector operator-(const Vector& other) const {
        return Vector(x - other.x, y - other.y);
    }
};

int main() {
    Vector v1(2,3), v2(1,4);
    Vector v3 = v1 + v2;  // calls operator+
    // v3.x = 3, v3.y = 7
}
<< Stream Operator ▼
⚠️ << must be a FRIEND function — left operand is ostream, not our class!
class Point {
public:
    int x, y;
    Point(int x=0, int y=0): x(x), y(y) {}
    
    // Must be friend — ostream is on the left
    friend ostream& operator<<(ostream& out, const Point& p) {
        out << "(" << p.x << ", " << p.y << ")";
        return out; // return ostream for chaining: cout << p1 << p2
    }
};

int main() {
    Point p(3, 4);
    cout << p;           // prints: (3, 4)
    cout << p << endl;   // chaining works!
}
== and != Operators ▼
class Student {
public:
    int id;
    string name;
    Student(int id, string n): id(id), name(n) {}
    
    bool operator==(const Student& other) const {
        return id == other.id;  // compare by id
    }
    
    bool operator!=(const Student& other) const {
        return !(*this == other); // reuse ==
    }
};

int main() {
    Student s1(1, "Ali"), s2(1, "Sara"), s3(2, "Bob");
    cout << (s1 == s2);  // 1 (true, same id)
    cout << (s1 != s3);  // 1 (true, different id)
}
++ Prefix and ++ Postfix ▼
Trick: Postfix takes a dummy int parameter to distinguish from prefix!
class Counter {
private:
    int count;
public:
    Counter(int c=0) : count(c) {}
    
    // PREFIX ++obj — increment THEN return
    Counter& operator++() {
        ++count;
        return *this;  // return reference to current
    }
    
    // POSTFIX obj++ — return THEN increment
    Counter operator++(int) {  // dummy int parameter!
        Counter temp = *this;  // save current state
        ++count;                 // then increment
        return temp;             // return old state
    }
    
    int getCount() const { return count; }
};

int main() {
    Counter c(5);
    cout << (++c).getCount();  // 6 (increment first)
    cout << (c++).getCount();  // 6 (return old), c is now 7
    cout << c.getCount();      // 7
}
= Assignment Operator ▼
⚠️ Always check for self-assignment! (this != &other)
class MyArray {
private:
    int* data;
    int size;
public:
    MyArray(int n) : size(n) { data = new int[n]; }
    
    // Copy assignment operator
    MyArray& operator=(const MyArray& other) {
        if (this != &other) {   // SELF-ASSIGNMENT CHECK
            delete[] data;      // free old memory
            size = other.size;
            data = new int[size]; // allocate new
            for(int i=0; i<size; i++)
                data[i] = other.data[i];
        }
        return *this;  // allows a = b = c chaining
    }
    
    ~MyArray() { delete[] data; }
};

Static Variables and Functions

Static members belong to the class, not to individual objects. Shared across ALL instances.

Static variable: one copy shared by all objects. Static function: can only access static members.
class Employee {
private:
    string name;
    static int count;  // shared by ALL Employee objects

public:
    Employee(string n) : name(n) {
        count++;  // increments shared counter
    }
    
    ~Employee() {
        count--;  // decrements when destroyed
    }
    
    // Static function — no 'this' pointer
    static int getCount() {
        return count;
        // cannot access 'name' here — it's non-static
    }
    
    void display() { cout << name << endl; }
};

// Must define static variable outside class!
int Employee::count = 0;

int main() {
    cout << Employee::getCount(); // 0
    {
        Employee e1("Ali");
        Employee e2("Sara");
        cout << Employee::getCount(); // 2
    } // e1, e2 destroyed here
    cout << Employee::getCount(); // 0
}

Tracing Output & Fixing Errors

Constructor/Destructor Order

Constructors: called in ORDER of creation. Destructors: called in REVERSE order (LIFO).
class Demo {
    string name;
public:
    Demo(string n) : name(n) { cout << "Ctor: " << name << endl; }
    ~Demo() { cout << "Dtor: " << name << endl; }
};

int main() {
    Demo a("A");
    Demo b("B");
    {
        Demo c("C");
    } // c destroyed here
    Demo d("D");
} // d, b, a destroyed (reverse)
Output:
Ctor: A
Ctor: B
Ctor: C
Dtor: C
Ctor: D
Dtor: D
Dtor: B
Dtor: A

Common Syntax Errors to Fix

Error 1: Missing semicolon after class ▼
// WRONG
class Box {
    int x;
}  // ← Missing semicolon!

// CORRECT
class Box {
    int x;
};  // ← Semicolon required!
Error 2: Accessing private member directly ▼
class A {
private: int x;
public: A(int v): x(v) {}
};
int main() {
    A obj(5);
    cout << obj.x;  // ERROR: x is private
    // Fix: add a getter
}
Error 3: Using delete instead of delete[] ▼
int* arr = new int[10];
delete arr;    // WRONG — undefined behavior!
delete[] arr; // CORRECT
Error 4: Forgetting const in copy constructor ▼
// WRONG
Student(Student& other) { ... }

// CORRECT — must be const reference
Student(const Student& other) { ... }
Error 5: Static member not defined outside class ▼
class A {
public:
    static int count; // declaration only!
};
// This line is REQUIRED outside the class:
int A::count = 0; // definition
